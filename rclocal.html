Сервис rc.local в Debian

Аннотация:  Добавление rc.local в последние версии Debian.
Дата:  Октябрь 2024.
Статус статьи:  pg_stat.


Сервис rc.local предназначен для автозапуска различных скриптов (или просто запуска каких-либо команд) при загрузке системы. В новых версиях дистрибутивов Debian этот сервис отсутствует. Поправим эту несправедливость! :)

Первым делом создадим сам сервис:

$ sudo nano /etc/systemd/system/rc-local.service

Записываем в файл следующее содержимое:

[Unit]
Description=/etc/rc.local
ConditionPathExists=/etc/rc.local

[Service]
Type=idle
ExecStart=/etc/rc.local start
TimeoutSec=0
StandardOutput=tty
RemainAfterExit=yes

[Install]
WantedBy=multi-user.target

Далее создаём сам файл rc.local:

$ sudo nano /etc/rc.local

И записываем дефолтную конфигурацию:

#!/bin/sh -e
#
# rc.local
#
# This script is executed at the end of each multiuser runlevel.
# Make sure that the script will "exit 0" on success or any other
# value on error.
#
# In order to enable or disable this script just change the execution
# bits.
#
# By default this script does nothing.

exit 0

Даём права на выполнение:

$ sudo chmod +x /etc/rc.local

Добавляем сервис в автозапуск:

$ sudo systemctl enable rc-local

Запускаем сервис:

$ sudo systemctl start rc-local

Смотрим состояние сервиса:

$ sudo systemctl status rc-local
